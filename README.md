# LogSystem 日志系统组件

> 基于 Lua 5.1 语法开发的轻量级日志输出组件，支持多级日志、时间戳、函数溯源及原型保护

## 功能特性

- ✅ **四级日志输出**：支持 `DEBUG` / `INFO` / `WARN` / `ERROR` 四种日志级别
- ✅ **可配置时间戳**：通过属性控制日志前缀是否显示系统时间（`时:分:秒` 格式）
- ✅ **函数溯源**：支持传入调用函数名称，便于日志定位与调试
- ✅ **错误分级输出**：`ERROR` 级别日志自动使用 `printError` 输出，其余级别使用 `print`
- ✅ **原型保护机制**：通过元表拦截，防止外部代码意外修改组件私有成员（`_` 开头）

---

## 组件属性

| 属性名 | 类型 | 默认值 | 显示名称 | 说明 |
| :-: | :-: | :-: | :-: | :-: |
| `ShowTimestamp` | `boolean` | `true` | 显示时间戳 | 开启后将在日志前显示当前系统时间（格式：`[HH:MM:SS]`） |

> 属性配置遵循组件框架规范，可在编辑器属性面板中可视化配置

## 🔧 开放函数

### `Debug(message, funcName?)`

输出调试级别日志，用于开发阶段的信息追踪

| 参数 | 类型 | 必填 | 说明 |
| :-: | :-: | :-: | :-: |
| `message` | `any` | ✅ | 日志内容，支持任意类型（自动转换为字符串） |
| `funcName` | `string` | ❌ | 调用该日志的函数名称，用于溯源定位 |

**返回值**：`nil`

---

### `Info(message, funcName?)`

输出信息级别日志，用于记录常规运行状态

| 参数 | 类型 | 必填 | 说明 |
| :-: | :-: | :-: | :-: |
| `message` | `any` | ✅ | 日志内容 |
| `funcName` | `string` | ❌ | 调用函数名称 |

**返回值**：`nil`

---

### `Warn(message, funcName?)`

输出警告级别日志，用于提示潜在问题或非阻断性异常

| 参数 | 类型 | 必填 | 说明 |
| :-: | :-: | :-: | :-: |
| `message` | `any` | ✅ | 日志内容 |
| `funcName` | `string` | ❌ | 调用函数名称 |

**返回值**：`nil`

---

### `Error(message, funcName?)`

输出错误级别日志，用于记录严重错误，自动使用 `printError` 输出

| 参数 | 类型 | 必填 | 说明 |
| :-: | :-: | :-: | :-: |
| `message` | `any` | ✅ | 日志内容 |
| `funcName` | `string` | ❌ | 调用函数名称 |

**返回值**：`nil`

---

## 内部实现说明

| 成员 | 类型 | 说明 |
| :-: | :-: | :-: |
| `_LEVELS` | `table` | 日志等级数值映射表（`DEBUG=1` ~ `ERROR=4`），用于级别比较 |
| `_OutputLog` | `function` | 内部日志格式化与输出核心函数，不对外暴露 |

> 所有以 `_` 开头的成员均为私有实现，外部调用请仅使用公开的 `Debug`/`Info`/`Warn`/`Error` 方法

---

## 注意事项

1. **依赖环境**：组件依赖 `os.date` 和 `string.format`，请确保运行环境支持标准 Lua 5.1 库
2. **消息类型**：`message` 参数支持任意类型，内部会自动调用 `tostring()` 转换，建议传入可序列化的内容
3. **性能建议**：高频调用日志时（如循环内），建议先判断日志级别或通过属性关闭输出，避免不必要的字符串拼接开销
4. **线程安全**：组件未内置锁机制，在多线程/协程环境下调用需自行保证并发安全
